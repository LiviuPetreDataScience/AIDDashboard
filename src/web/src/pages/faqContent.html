Get started

 

Selecting your project

 

Use the dropdown at the top of the page to select your project.

Edit Mode vs. View Mode

 

When you first select a project, you are in View mode - the data is read-only and the Submit button is grayed out. To make changes, click Edit Mode (or the Edit button on some pages).

 

Important: Use the Edit button below the project dropdown (circled in green below). Do not use the SharePoint “Edit” in the top-right corner (crossed out in red) - that edits the page itself, not your data.

💡 Tip: Zoom your browser to 75% to see the full page layout more easily.

Save and Submit

 

After making changes on any tab:

  1. 1.

    Click Save Edits to save your work.

  2. 2.

    Click Submit to commit your changes.

⚠ Note: Save Edits saves your data but does not finalize it. You must also click Submit. If you only save without submitting, your changes are not yet visible to the admin team.

Common rules (apply to all tabs)

 

  • Numbers without spaces - enter 1500, not 1 500.

  • Times in CET (Central European Time) - applies to Support Hours.

  • Complete all fields for any row you add - incomplete rows are flagged as errors in the admin dashboard.

  • Remember: Edit Mode → make changes → Save Edits → Submit - this applies to every tab.

Contractual languages

 

This section breaks down Support Engineers by country and language. It shows how many support engineers work at each location for each language.

 

This is a fixed grid - the languages and countries are predefined. You fill in numbers but cannot add or remove languages. This is different from the Automations tab where you can add rows.

 

Key rule: The total Support Engineers per country here must equal the Support Engineers number for that same country in the Latest Approved Staffing Model.

 

Example: If Romania shows 4.5 Support Engineers in the Latest Approved Model, then the sum of all language entries for Romania (e.g. 1.5 English + 3.0 German) must also equal 4.5.

 

Do not fill out the EMEA (Management and Specialists) row.

⚠ Note: This section is only required if your service type includes Service Desk. If your account has no SD services, you can skip this tab - but if SD is in scope, at least one language must have a value.

Countries/Users/Devices supported

 

This section applies to all service types, not just LTS. All countries where you provide support should be listed here.

To be completed by: Presales / Transition / Delivery (all). This is a shared responsibility, not just the service manager’s task.

 

Fields:

  • # Users - number of end users supported in each country. Must be at least 1.

  • # Devices - number of devices. Required where you have LTS (Local Technical Support, also known as Field Support). Must be at least 1.

  • # LTS Staff - number of local technical support engineers. Required if your service type includes LTS.

Services

 

The Services tab maps all services delivered on the account. This information is used for identifying upsell and cross-sell opportunities, so it must be accurate and current.

 

How to fill it in

 

  1. 1.

    Go to the Services tab and click Edit Mode.

  2. 2.

    Use the Filter by Category dropdown to narrow the list. A Subcategory filter appears for categories that have subcategories.

  3. 3.

    Fill in the required fields for each service.

  4. 4.

    Click Save Edits, then Submit.

Columns

 

 

  • Path - the service category path  List of categories  HERE

  • Services - the specific service name.

  • Contract end date (SOW) - the end date for this specific service’s contract (not the overall account contract).

  • Does this service exist in the company? - “company” here means the client’s environment, not Stefanini. Answer Yes if the client already has this service.

  • Is it provided by Stefanini Group? - Yes/No.

  • If not provided by SG, then by who? - select the competitor. If the competitor is not in the list, contact Emoke Guiu via email or Teams to have it added.

  • If service is not existent, is there an opportunity to provide it? - Yes/No. Marks a potential upsell.

  • Details - free-text for any relevant context.

 

To see the full service catalog, click the link provided at the top of the Services page.

⚠ Note: At least one service must be mapped per account.

Opportunities

 

The Opportunities tab is for tracking upsell and cross-sell efforts - new services, additional scope, or project-based work. This is separate from the Services tab.

 

 

 

  • Opportunity type - dropdown: New service, Additional scope, Project-based work.

  • Related service - select from services already mapped on the Services tab. If the service you need is not in this dropdown, go to the Services tab first and map it.

  • Status - current status of the opportunity.

  • Estimated Monthly Value ($) - projected monthly revenue in USD.

  • Contract Duration (Months) - expected duration.

  • Delivered by - which tower delivers this (same structure as “In Scope” on the Overall tab).

  • Details - free-text for context.

 

Important:

  • Only map opportunities where you have a contribution or input.

  • Renewals are out of scope for this page.

Support hours

 

This tab defines support coverage per language. All times are in CET (Central European Time).

 

 

  • Language - the support language (fixed list: EMEA English, NA English, EMEA French, German, Polish, etc.).

  • From (M-F) / To (M-F) - custom support hours in HH:MM format, Monday to Friday. Example: 8:00 to 22:30.

  • 24/5 - radio button. Select if support is 24 hours, Monday to Friday.

  • 24/7 - radio button. Select if support is 24 hours, 7 days a week.

  • On call / Interpret (DHS) - checkbox. Check if this language has on-call or interpretation services (typically for DHS accounts).

  • Sophie - checkbox. Check if the Sophie virtual assistant/chatbot is deployed for this language.

 

How to fill it in: For each supported language, either enter custom From/To hours or select 24/5 or 24/7. These are mutually exclusive - you use custom hours for partial coverage, or the radio buttons for full coverage.

✅ Before you submit - Support Hours

SLAs and KPIs

 

For each SLA or KPI, define what it measures, how it is calculated, and what the target is.

 

 

 

 

  • Name - identifier for the SLA/KPI (e.g. “SLA-3 / Backlog Aging”).

  • Type - dropdown: SLA or KPI.

  • Description - what it measures in plain language (e.g. “The average score of User Satisfaction survey responses”).

  • Formula - the calculation (e.g. “SUM(Survey response score) / SUM(Surveys)”). Enter “n/a” if no formula applies.

  • Target % - the threshold value. Format depends on direction (see below).

  • Measurement type - dropdown for the measurement method.

  • Can it be reported? - Yes/No. Whether this metric is included in reports.

  • Financial penalties - Yes/No. Whether a financial penalty applies if the target is not met.

  • Bonus - Yes/No. Whether a bonus applies if the target is met or exceeded.

Direction

 

The Target % field’s direction matters:

  • Less than (Low) - the result needs to be lower than the target to pass. Example: Backlog Aging target “Less than 5%” means keeping backlog under 5%.

  • Equal or more (High) - the result needs to be higher than the target to pass. Example: Client Satisfaction target “Equal or more than 90%”.
